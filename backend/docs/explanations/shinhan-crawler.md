# 신한카드 크롤러 설명

## 선정 이유

2026-06-20 기준 신한카드 공식 `robots.txt`는
`/pconts/html/card/` 경로를 일반 크롤러에 명시적으로 허용한다.

삼성카드는 일반 크롤러를 차단하고, KB국민카드와 현대카드는 일반 사용자
에이전트 규칙이 명확하지 않아 첫 실제 수집 채널에서 제외했다.

## 수집 경로

```text
공식 메인 페이지
https://www.shinhancard.com/pconts/html/main.html
  -> 공식 브라우저 검색 API에서 최대 15개 상품 URL과 연회비 발견
  -> API 실패 시 JSON-LD ItemList로 fallback
  -> /pconts/html/card/apply/credit/... 상세 페이지 요청
  -> Product JSON-LD, 혜택 본문, cardPlate 이미지 변수 파싱
  -> SQLite 및 로컬 이미지 저장
```

공식 검색 API는 신한카드 카드 목록 화면의 브라우저 스크립트가 직접 호출하는
공개 엔드포인트만 사용한다. 상품 URL과 대표 연회비 발견에만 사용하며, 혜택
원문과 이미지는 기존 공식 상세 페이지에서 수집한다. `/mob/` 경로는 사용하지
않는다.

## 수집 필드

- 카드명
- 신용카드 구분
- 상세 페이지 URL
- 공식 카드 이미지
- 혜택 제목과 원문
- 할인율 또는 정액 할인
- 파싱 가능한 최소 전월 실적
- 수집 HTML 스냅샷
- 검토 사유

## 현재 한계

연회비 값은 정적 상세 HTML의 템플릿에 표시 위치만 있고 실제 값은 페이지 로딩
후 공식 상세 API 데이터로 채워진다. 자동 크롤러는 정적 상세 HTML만 수집하므로
처음에는 `annual_fee=null`, `연회비 검토 필요` 상태로 저장한다.

Deep Oil은 공식 페이지 스크립트가 사용하는 공개 상세 조회 응답으로 연회비를
검증했다.

```text
상품 식별자: 201802130001
대표 연회비: 10,000원
브랜드별 총연회비: 13,000원, 10,000원
```

현재 추천 스키마는 단일 대표 연회비만 지원하므로 공식 응답의
`afeAmountOrigin=10000`을 사용한다. 출처 URL과 검증 시각을
`annual_fee_source_url`, `annual_fee_verified_at`에 저장한다. 재크롤링에서
정적 HTML이 연회비를 제공하지 않더라도 검증된 값과 활성 상태를 덮어쓰지 않는다.

Discount Plan+의 카드 전체 통합 할인 한도는 `CardBenefitTier`로 구조화했다.

```text
40만 이상 80만 미만: 3만원
80만 이상 120만 미만: 5만원
120만 이상 150만 미만: 7만원
150만 이상 180만 미만: 7만3천원
180만 이상: 10만원
```

Deep Oil의 표는 할인액 한도가 아니라 서비스별 이용금액·횟수 한도이므로 같은
카드 전체 구간 모델로 변환하지 않았다. 대신 생활서비스 공유 한도를
`CardServiceLimitTier`로 구조화했다.

```text
30만원 이상 70만원 미만:
  생활서비스 이용금액 한도 15만원
  5% 할인 한도 7,500원

70만원 이상:
  생활서비스 이용금액 한도 30만원
  5% 할인 한도 15,000원
```

카페와 편의점 혜택은 `life_service` 그룹으로 같은 한도를 공유한다.

혜택 업종을 확실히 분류하지 못하면 `etc`로 활성화하지 않고
`category_mapping` 검토 조건을 기록한다.

## 제한 수집 결과

2026-06-20 실행:

```text
python manage.py crawl_cards --issuer shinhan --limit 3
```

저장 결과:

- 신한카드 Discount Plan+
- 신한카드 Deep Oil
- 신한카드 Simple Plan+
- 혜택 규칙 7개
- 이미지 11개
- 크롤링 실패 0개

동일 명령을 다시 실행해 카드와 이미지가 중복 생성되지 않는 것을 확인했다.
원문 스냅샷은 수집 이력이므로 실행마다 추가된다.

## 15개 확대 수집 결과

2026-06-20 실행:

```text
python manage.py crawl_cards --issuer shinhan --limit 15
```

공식 검색 목록 상위 15개와 기존 Deep Oil을 합쳐 SQLite에 카드 16개를
보존했다.

- `active`: 1개
- `review_required`: 15개
- 공식 대표 연회비와 출처 저장: 16개
- 이미지 메타데이터: 52개
- 이미지 파일 저장 성공: 49개
- 상세 카드 수집 실패: 0개

이미지 실패 3개는 공식 CDN이 403으로 거부한 뒷면 GIF다. 해당 카드의 앞면
이미지는 정상 저장돼 추천 UI 표시에는 영향이 없다.

`신한카드 처음`은 공식 상품 URL과 pageId가 다른 2건이 검색 결과에 존재한다.
혜택이 유사하지만 공식 식별자가 다르므로 자동 병합하지 않고 중복 검토
대상으로 유지한다.

## 추천 활성 카드

공식 상세 원문에서 할인 대상과 월 한도를 명확하게 구조화할 수 있는 혜택만
선정해 다음 5개를 추천 후보로 활성화했다.

- 신한카드 Deep Oil
- 신한카드 Simple Plan+
- 배민 신한카드 밥친구
- 알리익스프레스 신한카드
- 신한카드 Edu Plan+

카드별 활성 범위:

```text
Simple Plan+: 국내 이용금액 1.5%
배민 밥친구: 배달의민족 5%, 월 3만원
알리익스프레스: 알리익스프레스 10%, 실적별 월 1·2·3만원
Edu Plan+: 3대 할인점 1%, 전월 50만원 이상 월 2천원
Deep Oil: GS25·CU·스타벅스·이디야 5%, 생활서비스 공유 한도
```

그 밖의 복잡한 혜택은 카드가 제공하더라도 현재 추천 금액에는 포함하지 않는다.
지원 범위가 카드 전체 혜택의 일부인 경우 API와 검토 사유에서 이를 명시한다.

## 추천 사용 조건

수집 카드는 다음 조건을 충족한 뒤에만 `active` 상태로 전환한다.

1. 연회비 검증
2. 실적 구간별 할인 한도 구조화
3. 카테고리 매핑 검증
4. 건당·월 한도 검증
5. 상품설명서와 계산 결과 대조

## 활성화 검사 결과

다음 명령으로 카드별 활성화 가능 여부를 검사한다.

```text
python manage.py validate_cards
python manage.py validate_cards --card-id 2 --activate
```

Deep Oil은 공식 연회비 검증과 가맹점 거래 매칭 구현 후 `active`로 전환했다.
카페와 편의점 혜택만 계산하며, 아직 미지원인 주유·차량 혜택은 추천 금액에서
제외하고 `partial_benefit_coverage` 경고로 유지한다.
