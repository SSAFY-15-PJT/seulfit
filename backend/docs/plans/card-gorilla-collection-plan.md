# 카드고릴라 상위 카드 원문 수집 계획

## 목적

신용카드와 체크카드 추천 결과를 분리해 제공할 수 있도록 카드고릴라 공개
순위에서 각 20개 상품의 상세 원문을 확보한다.

카드고릴라는 후보 상품을 빠르게 확보하는 보조 출처로 사용한다. 최종 정규화와
활성화 과정에서는 카드사 공식 상품 설명서 또는 공식 상품 페이지를 우선 검증
출처로 사용한다.

## 수집 범위

- 신용카드 주간 순위 상위 20개
- 체크카드 주간 순위 상위 20개
- 카드명, 카드사, 신용/체크 구분
- 카드 이미지 원본 URL
- 연회비 원문
- 전월 실적
- 혜택 분류와 상세 원문
- 순위 및 상세 데이터 출처 URL

카드고릴라 순위와 점수는 SeulPick 추천 점수에 사용하지 않는다.

## 단계

1. `robots.txt`와 공개 API 접근 정책을 확인한다.
2. 신용카드와 체크카드 순위 API를 각각 조회한다.
3. 상세 API 응답을 `CrawlItem.raw_payload`에 저장한다.
4. 기존 `CardProduct`와 카드사, 카드명, 카드 종류 기준으로 중복을 확인한다.
5. 카드사 공식 자료로 혜택 조건을 검증한다.
6. 검증된 데이터를 정규화하고 SQLite의 `CardProduct`에 반영한다.
7. 전체 카드 정규화가 끝난 뒤 별도 승인 후 Neo4j를 동기화한다.

## 중단 복구

수집 상태는 `CrawlJob`과 `CrawlItem`에 카드 단위로 저장한다. 실행이 중단되면
다음 명령으로 완료되지 않은 항목부터 다시 시작한다.

```text
python manage.py crawl_cards --issuer card_gorilla --resume
python manage.py crawl_cards --issuer card_gorilla --resume --retry-failed
```
