# 카드 정규화 검수 현황

## 목적

수집된 카드가 추천 계산에 들어가기 전에 카드 유형과 혜택 조건이 충분히
구조화됐는지 확인한다. `review_required`는 수집 실패가 아니라, 원문은
보존됐지만 계산에 필요한 조건이 아직 완전히 정규화되지 않았다는 뜻이다.

## 2026-06-20 현황

SQLite에 저장된 카드 상품은 총 40개다.

| 카드사 | 신용카드 | 체크카드 | 합계 |
| --- | ---: | ---: | ---: |
| 신한카드 | 16 | 0 | 16 |
| 우리카드 | 12 | 3 | 15 |
| 토스뱅크 | 2 | 3 | 5 |
| 현대카드 | 4 | 0 | 4 |
| 합계 | 34 | 6 | 40 |

카드명에 `체크` 또는 `CHECK`가 있지만 `credit`으로 저장된 충돌 사례는
현재 0개다.

## 카드 유형 판별

- 신한카드: 현재 수집 경로가 신용카드 상품 경로이므로 `credit`
- 우리카드: 공식 상품 본문의 `체크카드` 표기를 기준으로 구분
- 토스뱅크: 상품 URL 목록의 신용카드 여부와 실제 발급사를 기준으로 구분
- 현대카드: 현재 수집 대상이 신용카드 상품이므로 `credit`

검증 단계에서는 카드명에 `체크` 또는 `CHECK`가 있는데 `card_type=credit`인
경우를 유효하지 않은 데이터로 처리한다. 반대로 카드명에 체크 표기가 없다는
이유만으로 신용카드라고 단정하지는 않는다.

## 검수 필요 카드 분류

현재 40개 중 `active`는 27개, `review_required`는 12개,
`inactive`는 1개다.

| 원인 | 카드 수 | 처리 방향 |
| --- | ---: | --- |
| 구조화된 혜택 없음 | 10 | 공식 혜택 원문을 카드별 규칙으로 추가 |
| 카테고리 또는 가맹점 조건 미지원 | 10 | 지원 카테고리와 가맹점 범위로 분해 |
| 출처 기반 추가 검수 필요 | 7 | 할인율, 한도, 실적 조건을 원문과 대조 |
| 요일 또는 시간 조건 미지원 | 2 | 조건 스키마 확장 후 활성화 |

한 카드에 원인이 여러 개 있을 수 있으므로 원인별 수의 합은 25보다 클 수 있다.

## 즉시 활성화하지 않은 이유

검수 대상 중 일부는 계산 가능한 혜택 규칙을 포함하지만, 원문의 월 이용 횟수,
실적별 한도 또는 가맹점 범위가 누락돼 있다. 예를 들어 신한카드 Unboxing의
정액 혜택은 `최대 6회` 조건이 있으므로 횟수 한도 없이 활성화하면 할인액을
과대 계산한다.

따라서 원문에 `최대 N회`가 있는데 `monthly_usage_limit`이 비어 있으면
`usage_limit_mapping` 사유로 `review_required`를 유지한다.

## Unboxing 정규화 결과

신한카드 Unboxing은 다음 조건을 구조화하고 `active`로 전환했다.

- 전월 실적: 40만원 이상
- 10% 캐시백: 전월 40만/80만/120만원 구간별 월 1만/2만5천/5만원
- 언박싱 스페셜: 3만원 미만 거래 건당 2,500원
- 스페셜 제공 횟수: 전월 40만/80만/120만원 구간별 월 2/4/6회
- 거래 상한: `maximum_transaction_amount=30000`

전월 실적 50만원 예시에서는 10% 캐시백 1만원과 스페셜 할인 5천원이
각각의 서비스 한도로 제한돼 월 총 1만5천원으로 계산된다.

## 신한카드 처음 정규화 결과

공식 식별자가 다른 동일 이름 상품 2개는 혜택·출시일이 같고 카드 디자인만
다르다. `1229296_2207`을 대표 상품으로 활성화하고, 구형 디자인
`1227020_2207`은 재수집 후에도 `inactive`를 유지한다.

현재 서비스 카테고리와 직접 연결되는 `오늘도 5% 적립`을 다음과 같이
구조화했다.

- 대상: 음식점, 카페, 편의점, 쿠팡·컬리
- 거래당 적용 금액: 최대 1만원
- 거래당 최대 적립: 500포인트
- 일 통합 적립 한도: 1,000포인트
- 월 통합 한도: 전월 30만/50만/100만원 구간별 5천/1만/2만포인트

일 한도는 카테고리별이 아니라 `오늘도` 서비스 전체에 공유 적용한다.
거래 데이터에 `transaction_date`가 없으면 일 한도를 검증할 수 없으므로
해당 혜택을 0원 처리한다.

정기결제, 소비관리 보너스 등 현재 카테고리 밖의 혜택은 추천 금액에서
제외했다.

## 우리카드 검수 결과

우리카드 본문의 유의사항에 포함된 `체크카드` 문구 때문에 일부 신용카드가
체크카드로 잘못 분류된 문제를 교정했다. 상품명의 `체크` 또는 `CHECK`를
우선하고, 그 외에는 공식 `카드 종류` 영역을 사용한다.

교정 결과 우리카드는 신용카드 12개, 체크카드 3개다.

### 카드의정석 SHOPPING+

- 오프라인 대상점 10% 할인
- 거래당 최대 할인 5,000원
- 전월 30만/70만/120만원 구간별 월 6천/1만2천/2만4천원 한도

온라인 쇼핑과 간편결제 추가 할인은 저장 원문의 대상 목록이 불완전해 추천
금액에서 제외했다.

### 카드의정석 오하CHECK

- SHOPPING: 쿠팡, 무신사, 지그재그, 네이버페이, 우리WON페이
- EAT: 스타벅스, 배달의민족, 쿠팡이츠, B마트, 마켓컬리
- 캐시백률 5%, 거래당 최대 1,000원
- 그룹별 전월 20만/50만/70만원 구간별 월 2천/4천/6천원 한도

PLAY·LIFE 혜택은 현재 추천 카테고리 범위 밖이므로 제외했다.
온라인·오프라인 전용 혜택은 거래의 `channel`이 없으면 계산하지 않는다.

### 검수 유지 상품

- 국민행복카드S2: 복합 업종과 서비스별 통합 한도 추가 정규화 필요
- 국민행복 체크카드S2: 복합 업종과 서비스별 통합 한도 추가 정규화 필요
- GS칼텍스 화물복지카드: 일 횟수와 주유 단위 혜택 처리 필요
- 카드의정석 EVERY MILE SKYPASS: 마일리지 단위를 금액 점수와 분리해야 함

### 카드의정석2 칼퇴 CHECK

현재 카테고리와 연결되는 저녁 음식점·카페·편의점 혜택을 활성화했다.

- 적용 시간: 18:00 이상 23:00 미만
- 음식점: 3만원 이상, 건당 최대 3천원, 일 1회, 월 3회
- 카페: 1만원 이상, 건당 최대 2천원, 일 1회, 월 3회
- 편의점 CU: 1만원 이상, 건당 최대 2천원, 일 1회, 월 2회
- 칼퇴 그룹 월 한도: 전월 30만/60만/120만원 구간별
  4천/8천/1만5천원

거래일자나 거래시각, 오프라인 채널 정보가 없으면 해당 혜택을 계산하지 않는다.

마일리지와 M포인트를 임의로 1원으로 환산하지 않는다. 환산율과 출처가
확정된 뒤 보상 단위 모델을 별도로 추가한다.

상태 전환은 카드별 테스트에서 할인액과 한도 적용 결과가 확인된 뒤 수행한다.
